# cn-official-guide

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件的形式存放，不针对任何单个域名或具体网站。

## 项目简介

本仓库的主要目的是提供一个结构化的存放空间，用于整理和发布各类独立的 HTML 资源。每个页面均为独立的文件，彼此之间无依赖关系。仓库本身不提供动态服务，仅作为静态页面的存储和分发点。

## 目录说明

仓库根目录下包含以下内容：

- `index.html`：仓库的入口页面，用于导航和说明。
- `pages/`：存放所有独立 HTML 页面的目录。每个页面为一个独立的 `.html` 文件。
- `assets/`：存放页面所需的共用资源，例如样式表、图片、字体等。
- `README.md`：本说明文件。

## 页面归档说明

- 所有页面均为静态 HTML，不包含服务器端逻辑。
- 每个页面在 `pages/` 目录下以独立文件形式存在，文件名应具有描述性，便于识别。
- 页面之间不应互相引用，保持独立性。
- 归档内容可能不定期更新，但不会影响已有页面的访问。

## 维护说明

- 本仓库由维护者根据需要进行更新。
- 如需提交新的页面或修改现有页面，请发起 Pull Request。
- 页面内容应符合通用发布规范，不包含非法、侵权或敏感信息。
- 维护者保留对页面内容进行审核和调整的权利。

## 使用方式

可通过直接访问 `pages/` 目录下的 HTML 文件来查看对应页面。例如，通过浏览器打开 `https://<你的仓库地址>/pages/example.html` 即可查看名为 `example.html` 的页面。
